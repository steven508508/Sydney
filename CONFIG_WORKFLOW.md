# CONFIG_WORKFLOW.md - 修改設定檔標準流程

修改設定檔（`openclaw.json`、`models.json`、或其他設定檔）前，必須嚴格遵守以下五步驟。

## 五步驟（強制執行）

### 1. 查證
先去官網或文件確認指令、語法是否正確，避免使用過時或錯誤的設定方式。

### 2. 驗證
用 Python 驗證 JSON 語法格式正確：
```bash
python3 -c "import json; json.load(open('檔案路徑'))"
```

### 3. 評估風險
- **會出什麼錯？** — 格式錯誤、服務啟不起來、功能失效
- **影響什麼？** — 哪些功能會受影響
- **如何復原？** — 確認有備份，隨時可以還原

### 4. 請求確認（重大操作）
重大操作前必須向用戶確認才能繼續。

### 5. 備份設定檔
執行任何修改前，先備份原檔案：
```bash
cp openclaw.json openclaw.json.$(date +%Y%m%d_%H%M%S).bak
```

---

## 重大操作定義

以下操作視為重大，**必須經過完整五步驟**：

| 操作 | 風險 |
|------|------|
| 修改設定檔（`openclaw.json`、`models.json` 等） | 設定錯誤導致服務無法啟動 |
| 重啟服務（Gateway、systemd 等） | 服務短暫中斷 |
| 刪除資料（記憶體、記錄檔、workspace 檔案） | 資料永久消失 |
| 安裝/移除 plugins | 可能影響系統穩定性或安全性 |

**低風險操作**（可快速執行，但仍要先查證+驗證）：
- 讀取設定檔
- 查閱日誌檔
- 檢查系統狀態

## tools.profile 警告說明

執行 `openclaw gateway` 時出現以下警告是**正常的，無需處理**：

> `tools.profile (coding) allowlist contains unknown entries (apply_patch)`

原因：`coding` profile 包含一些需要特定 API 或 runtime 才能使用的工具，在當前環境中這些工具無法使用，但**不影響任何正常功能**。這不是錯誤，只是資訊性警告。

## 快速檢查清單

每次修改設定檔前，快速確認：

- [ ] 我查證過指令/語法是正確的嗎？
- [ ] JSON 語法驗證通過了嗎？
- [ ] 我評估過風險並知道如何復原嗎？
- [ ] 這是重大操作嗎？需要的話有獲得確認嗎？
- [ ] 備份了嗎？

## 對外發布規則

### 平台信任分級

| 平台類型 | 範例 | 發布前確認 |
|---------|------|-----------|
| 🌐 公開平台 | Moltbook、Twitter/X、Discord 公開伺服器、論壇 | ✅ 永遠需要 |
| 💬 半公開平台 | Discord 私人伺服器、公開 Telegram 群組 | ✅ 永遠需要 |
| 🔒 私人平台 | Telegram 私聊、Signal、Email（私人） | ⚠️ 簡化確認 |
| 📁 內部工具 | Obsidian、Notion（私人）、本地腳本 | ❌ 不需要 |

### 對外發布工作流（強制）

```
1. 撰寫草稿
2. 執行敏感內容掃描
3. 🚨 發現敏感內容 → 停止，通知用戶，不發送
4. ✅ 乾淨草稿 → 展示給用戶
5. 📨 用戶回覆「可以」→ 正式發布
6. 📊 發布結果回報用戶
```

### 禁止發布的內容

以下內容**永不發布**，發現時立即停止並通知用戶：

| 類別 | 範例關鍵字 |
|------|-----------|
| API Key / Token | `moltbook_sk_`、`Bearer `、`api_key`、`secret` |
| 密碼相關 | `password`、`passwd`、`pwd`、`credentials` |
| 私人證件 | 身份證字號、護照號碼 |
| 聯絡方式 | 私人電話（09xx開頭）、私人地址 |
| 未授權的對話 | 不屬於公開資訊的私人對話截圖 |
| 主機路徑 | 含真實 IP、網段、內網域名 |

### 敏感內容掃描（自動執行）

每次對外發布前，自動檢查草稿是否含有：
- API key 格式（`sk_`、`moltbook_sk_` 等）
- Bearer token 格式
- 以 `-----BEGIN` 開頭的 PEM 私鑰
- 大量數字（疑似電話、證件）
- 非預期的 URL（不在預期清單內）
