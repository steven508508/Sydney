# HEARTBEAT.md

## 資源清理規則

每次任務完成後，自覺清理殘留資源：
- 後台進程（pkill 掉閒置的 Xvfb/VNC 等）
- 暫存檔案（/tmp、~/.openclaw/workspace/tmp*）
- Agent-browser 分頁（用完後關閉）
- 產出的圖檔、PDF 等（傳送後立即詢問是否刪除）

**主動告知原則：**
- 任務完成後，主動告知用戶已釋放多少記憶體
- 若有檔案產出，主動詢問是否刪除
- 格式：「✅ [任務] 完成，已釋放 XX MB 記憶體，暫存檔已清除」

## API 錯誤應對規則

當遇到 API 錯誤（rate limit、timeout、401/403/429/500 等）時，主動說明以下資訊：

1. **是哪個服務？**（Brave / Minimax / Veryfi / Gmail / 其他）
2. **具體限制是什麼？**（每分鐘/每秒幾次請求）
3. **預計多久恢復？**（冷卻時間）
4. **我會不會受影響？**（這次請求是否需要重試）

## 提醒規則

- 遇到 rate limit → 說明是哪個服務 + 等多久
- 遇到 401/403 → 可能是憑證過期，需要重新認證
- 遇到 429 → 通常是短暫的，稍後重試即可
- 遇到 500/503 → 服務端問題，無法解決只能等待

## 一般檢查

如果沒有特別需要處理的事項，回覆 HEARTBEAT_OK 即可。
